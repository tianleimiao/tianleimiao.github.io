# GitHub Pages Deploy

## Architecture

```
web/ (Next.js) → npm run build → web/out/ → GitHub Actions → Pages
```

Workflow: `.github/workflows/deploy-github-pages.yml`

## basePath logic (`web/next.config.ts`)

| Scenario | basePath |
|----------|----------|
| User site (`user.github.io`) | empty |
| Project site (`user.github.io/repo`) | `/repo` |
| Local dev | empty (no GITHUB_REPOSITORY) |

CI sets `GITHUB_REPOSITORY=owner/repo` during build.

Client code uses `process.env.NEXT_PUBLIC_BASE_PATH` for asset paths if needed.

## Local verification

Simulate project-site build:

```bash
cd web
set GITHUB_REPOSITORY=owner/repo-name   # Windows cmd
# or: $env:GITHUB_REPOSITORY="owner/repo-name"  # PowerShell
npm run build
```

Check `web/out/index.html` asset paths include `/repo-name/` prefix when expected.

## Deploy steps

1. Push to `main` or `master`
2. GitHub Actions runs `npm ci && npm run build` in `web/`
3. Uploads `web/out` as Pages artifact
4. Repo Settings → Pages → Source: **GitHub Actions**

## Common issues

| Symptom | Fix |
|---------|-----|
| CSS/JS 404 on Pages | basePath mismatch; rebuild with GITHUB_REPOSITORY |
| Works locally, broken on Pages | Test build with env var above |
| Shader blank on Pages | WebGL needs HTTPS (Pages provides); check console |
| Build fails in CI | Run `npm run validate` locally first |

## Manual trigger

Actions tab → "Deploy GitHub Pages" → Run workflow

## Do not commit

- `web/out/` — generated by CI
- `web/.next/` — build cache
